# aiwiki

[![CI](https://github.com/tommy5dollar/aiwiki/actions/workflows/ci.yml/badge.svg)](https://github.com/tommy5dollar/aiwiki/actions/workflows/ci.yml)
[![npm](https://img.shields.io/npm/v/aiwiki)](https://www.npmjs.com/package/aiwiki)
[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](LICENSE)

Generate Markdown documentation wikis from any codebase using OpenAI Codex CLI agents.

aiwiki uses a two-phase approach: first it plans a catalog of wiki pages by exploring your repo, then generates each page with a dedicated agent that reads the relevant source files in depth. The result is a set of interconnected Markdown files you can drop into any wiki, GitHub Pages site, or docs folder.

## GitHub Action

The easiest way to use aiwiki is via the composite GitHub Action. Add this workflow to any repository and trigger it manually — it will generate docs and open a pull request for review.

```yaml
# .github/workflows/generate-docs.yml
name: Generate Docs
on:
  workflow_dispatch:
    inputs:
      model:
        description: Codex model
        default: gpt-5-mini
      reasoning_effort:
        description: Reasoning effort (low, medium, high)
        default: low

permissions:
  contents: write
  pull-requests: write

jobs:
  generate:
    runs-on: ubuntu-latest
    steps:
      - name: Generate documentation
        uses: tommy5dollar/aiwiki@main
        with:
          openai_api_key: ${{ secrets.OPENAI_API_KEY }}
          model: ${{ inputs.model }}
          reasoning_effort: ${{ inputs.reasoning_effort }}
```

The action checks out your repository, runs the agents, then commits the generated docs to a `docs/autogenerated` branch and opens (or updates) a pull request targeting your default branch.

**Required permissions**: the workflow must grant `contents: write` and `pull-requests: write`.

### Action inputs

| Input | Default | Description |
|---|---|---|
| `openai_api_key` | — | **Required.** OpenAI API key used by Codex CLI |
| `openai_base_url` | `https://api.openai.com/v1` | OpenAI-compatible base URL (for proxies) |
| `model` | `gpt-5-mini` | Codex model |
| `reasoning_effort` | `low` | Reasoning effort: `low`, `medium`, `high` |
| `output_dir` | `docs/autogenerated` | Output directory (relative to repo root) |
| `concurrency` | `3` | Number of concurrent page agents |
| `page_timeout` | `300000` | Per-page timeout in ms |
| `timeout` | `1200000` | Overall timeout in ms |
| `excluded_dirs` | | Comma-separated extra directories to exclude |
| `project_name` | basename of repo | Project name used in prompts |
| `trace_id` | auto-generated | Trace ID to tag this run |
| `log_level` | `info` | Log level: `debug`, `info`, `warn`, `error` |

### Action outputs

| Output | Description |
|---|---|
| `output_dir` | Resolved output directory where docs were written |
| `trace_id` | Trace ID used for this run |
| `pr_url` | URL of the created or updated pull request |

## CLI

aiwiki can also be run directly from the command line.

### Prerequisites

- Node.js >= 18
- [OpenAI Codex CLI](https://github.com/openai/codex) installed globally (`npm install -g @openai/codex`)
- `OPENAI_API_KEY` environment variable set

### Installation

```bash
npm install -g aiwiki
```

### Usage

```bash
aiwiki --repo-root /path/to/repo
```

Or run inside the repo root with no flags:

```bash
cd /path/to/repo
OPENAI_API_KEY=sk-... aiwiki
```

### Options

| Flag | Env Var | Default | Description |
|---|---|---|---|
| `--repo-root <path>` | `DOCS_GEN_REPO_ROOT` | `process.cwd()` | Repository root directory |
| `--model <model>` | `DOCS_GEN_MODEL` | `gpt-5-mini` | Codex model |
| `--reasoning-effort <level>` | `DOCS_GEN_REASONING_EFFORT` | `low` | Reasoning effort: low, medium, high |
| `--output-dir <dir>` | `DOCS_GEN_OUTPUT_DIR` | `docs/autogenerated` | Output directory (relative to repo root) |
| `--concurrency <n>` | `DOCS_GEN_CONCURRENCY` | `3` | Number of concurrent page agents |
| `--page-timeout <ms>` | `DOCS_GEN_PAGE_TIMEOUT` | `300000` | Per-page timeout (ms) |
| `--timeout <ms>` | `DOCS_GEN_TIMEOUT` | `1200000` | Overall timeout (ms) |
| `--excluded-dirs <dirs>` | `DOCS_GEN_EXCLUDED_DIRS` | `node_modules,.git,...` | Comma-separated directories to exclude |
| `--project-name <name>` | `DOCS_GEN_PROJECT_NAME` | basename of repo root | Project name used in prompts |
| `--trace-id <id>` | `DOCS_GEN_TRACE_ID` | auto-generated | Trace ID to tag this run |
| `--log-level <level>` | `DOCS_GEN_LOG_LEVEL` | `info` | Log level: debug, info, warn, error |

### Configuration file

Place an `.aiwiki.json` in the root of the repo being documented. CLI flags take priority over this file.

```json
{
  "projectName": "my-project",
  "model": "gpt-5-mini",
  "reasoningEffort": "low",
  "excludedDirs": "node_modules,.git,dist,coverage"
}
```

Config priority: CLI flags > environment variables > `.aiwiki.json` > defaults

## Output

Generates a directory containing:
- `index.md` — table of contents with page priorities
- `{slug}.md` — one Markdown page per wiki topic

## Validation

aiwiki automatically post-processes generated output:
- Auto-fixes Mermaid node labels containing special characters (`()`, `{}`, `|`, `[]`, `/`, `\`)
- Validates Mermaid diagram syntax via `mmdc`
- Checks minimum page length and heading structure
- Verifies link integrity between index and pages
- Detects orphaned pages

## Programmatic API

```typescript
import {
  loadConfig,
  generateCodexWiki,
  generateIndex,
  validateOutput,
  reportValidation,
  writeOutput,
} from 'aiwiki';
import type { Config, WikiStructure, GeneratedPage } from 'aiwiki';

const config = loadConfig();
const { structure, pages } = await generateCodexWiki(config);
const indexContent = generateIndex(structure);
const issues = validateOutput(structure, indexContent, pages);
reportValidation(issues);
writeOutput(config, indexContent, pages);
```
